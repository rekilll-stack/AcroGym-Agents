# n8n — гид владельца (без техники)

Что это: **n8n — конвейер приёма заявок**. Сайт шлёт заявку → n8n проверяет
токен, чистит телефон, присваивает Lead UID → пишет строку в Google-таблицу
«Leads» (витрина) → lead-helper забирает её и шлёт карточку в Assistant-бот.

---

## Как зайти в редактор n8n

1. Открой **https://hook.acrogym.org/**
2. Браузер спросит логин/пароль (серое системное окно) — это **первая дверь
   (nginx basic-auth)**: логин `acrogym`, пароль — тот, что ты сам задавал
   при настройке nginx.
3. Откроется страница входа n8n — это **вторая дверь**: email
   `rekilll@gmail.com`, пароль — тот, что ты сам задал 2026-06-11 при настройке
   owner-аккаунта. Он есть только у тебя, в файлах на сервере его нет.
   Потеряешь — восстановление: `cd /home/admin/n8n && sudo docker compose exec n8n n8n user-management:reset`,
   затем заново задать пароль через браузер (воркфлоу и доступы сброс не трогает).

## Как посмотреть, что лиды проходят (Executions)

1. В левом меню n8n → **Executions** (часики).
2. Каждая строка = одна заявка с сайта:
   - 🟢 **Succeeded** — лид прошёл, строка в таблице есть;
   - 🔴 **Error** — упал. Кликни по строке: откроется схема, красный узел =
     где именно сломалось (чаще всего Google Sheets — права/квота).
3. Самый простой повседневный контроль — даже не сюда, а просто **смотреть
   таблицу «Leads»**: новая заявка = новая строка в течение секунд, карточка
   в Assistant — в течение минуты.

## Если лиды не приходят — диагностика по шагам

1. **Таблица «Leads»**: строка от заявки появилась?
   - Появилась, а карточки нет → проблема на стороне бота (lead-helper), не n8n.
   - Не появилась → шаг 2.
2. **Executions в n8n**: есть запуск от этого времени?
   - Есть, красный → кликни, посмотри какой узел красный, сфоткай — разберём.
   - Нет вообще → заявка не дошла до n8n (сайт не шлёт / токен / nginx) → шаг 3.
3. **Жив ли n8n** — в терминале сервера:
   ```bash
   curl https://hook.acrogym.org/healthz
   ```
   Ответ `{"status":"ok"}` = n8n жив, ищи проблему на стороне сайта (пусть
   сайтщик прогонит свои curl-тесты из docs/website-form-integration.md).
4. **Крайняя мера — перезапуск n8n** (нужен sudo, твой пароль):
   ```bash
   cd /home/admin/n8n && sudo docker compose restart
   ```
   Подождать ~30 секунд, повторить curl из шага 3.
5. **Если труба лежит, а лиды идут** — верни бота на старую Google-форму:
   см. `docs/rollback-lead-source.md` (2 минуты). Сайт-заявки за время простоя
   n8n будут потеряны — поэтому это именно аварийный режим, чинить всё равно надо.

## Где что лежит: /home/admin/n8n/

```
docker-compose.yml    — как запускается n8n (НЕ редактировать без нужды)
.env                  — СЕКРЕТЫ: ключ шифрования + webhook-токен. НЕ ТРОГАТЬ.
nginx/                — копии конфигов nginx (рабочие — в /etc/nginx)
workflows/            — JSON-копия воркфлоу (бэкап; рабочая версия внутри n8n)
website-dev-handoff.txt — webhook-токен для сайтщика (передавать лично)
```

## 🔴 ВАЖНО — N8N_ENCRYPTION_KEY

В файле `/home/admin/n8n/.env` лежит `N8N_ENCRYPTION_KEY`.
**НИКОГДА не менять и не терять.** Этим ключом зашифрованы все доступы внутри
n8n (Google-таблица, webhook-токен). Сменишь или потеряешь — n8n не сможет их
расшифровать, труба умрёт, и доступы придётся пересоздавать с нуля.
Он же — причина, почему файл `.env` вообще нельзя трогать.
